02 · Architecture

Strict plane separation

Control Plane · Cloud (Gemini et al.)

Structural metadata only

  • Empty column names + dtypes
  • Abstract research task (“groupby GPA by program”)
  • Returns code logic only
  • Never: PII, rows, linkage keys, joint demo×outcome schemas without break-glass
Data Plane · Campus hardware

Sealed local execution

  • Human-in-the-Loop attestation before run
  • pandas · numpy · scipy (local open source)
  • No outbound analysis egress in PoC design
  • Outputs: aggregates under university custody
Assertion: Cloud sees structure. Campus sees subjects. Never the reverse.

04 · Confidentiality Engineering

Separate · unlink · synthesize

Ordinary use (no join)

  • Independent opaque ID namespaces (DEM_* vs KEY_*)
  • Independent row shuffles — destroy alignment
  • Zero shared columns across demo ↔ key tables
  • PII vault held apart from both analysis tables

Synthpop-style copies

  • Marginal-preserving categorical resample
  • Rank-swap + scaled noise on GPA / ETS
  • Communicates trends for AOL discussion
  • Defeats casual recombination / exact-row recovery
Sealed linkage_vault.json can reconstruct joins — but only after break-glass authorization is logged. Ordinary analysts never receive the vault.

07 · Break-Glass

Equity linkage is exceptional

Default rule

  • Demographics × key outcomes are blocked
  • Ordinary AOL monitoring uses key_academic alone
  • Control Plane joint schemas are withheld

Authorization record (local JSONL)

  • timestamp_utc
  • approver (name/role)
  • reason
  • analysis_requested
  • tables_linked
  • authorization_id (UUID)
