03 · Architecture

Where packages and plots may run

Control Plane · Cloud

May receive

  • Empty schemas (column names / dtypes)
  • “Draft a seaborn heatmap of gpa vs ets_score”
  • Returned code snippets only

Must not receive

  • CSV / frame rows / chart image pixels of real data
  • Disney/real names, emails, student IDs
  • Joint demo×outcome schemas (pre–break-glass)
Data Plane · Campus

Runs locally

  • pandas, numpy, scipy — analysis
  • matplotlib, seaborn — figures in this deck
  • Human-in-the-Loop gate before execution
  • Figures retained under university custody
PoC script: scripts/decoupled_ai_protocol_poc.py

06 · Not acceptable by default

Demo × outcome visuals — refuse unless break-glass

These figures require linking demographics to academic outcomes. Without a logged break-glass authorization, IT should treat them as not approved — and Control Plane must not receive the joint schema.

Bar — GPA by gender (linked)
Status: NOT ACCEPTABLE without break-glass log
Heatmap — demographics × GPA/ETS
Status: NOT ACCEPTABLE by default
Even after break-glass: PII vault stays sealed; AI still gets empty schema only — never the chart data or names/emails.

09 · Break-glass

Only path to demo × outcome charts

Required log fields

  • timestamp_utc
  • approver (name/role)
  • reason
  • analysis_requested
  • tables_linked
  • authorization_id

Still forbidden after approval

  • Uploading linked rows to Gemini/OpenAI
  • Including PII vault columns in extracts
  • Skipping Human-in-the-Loop attestation
  • Treating synthpop tables as “anonymous enough” to send off-campus
Break-glass unlocks local linkage. It does not authorize cloud ingestion.
